Stage Projection 必须对 Workflow 定义零假设

Web 不知道应该有几个 Stage，也不知道它们叫什么。它只接收一个带顺序、状态与历史锚点的投影数组，并按密度自动选择展示方式。

Workflow / Bridge 返回

stage_projection: {
  revision: "workflow-v7",
  items: [
    {
      key, label, status,
      checkpoint_id,
      summary?
    }
  ]
}
→

Web 通用渲染器只负责

  • 保持 Workflow 给出的顺序与文字
  • 根据 items 数量切换显示密度
  • 当前项突出、其余项降权
  • 历史项链接精确 checkpoint_id
  • 不提供 setStage 或任何状态写入
