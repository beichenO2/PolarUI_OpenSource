## 极限目标

**一句话**：PolarUI workflow 可通过双入口（PolarUI Web 按钮 + CLI）一键编译导出为独立 LibreChat 网站发行版；`npm run test:web-release` 全绿，覆盖 `WEB_ACCEPTANCE.md` 全部 AC 用例，无需人工点验。

**可度量验收**：
1. `node scripts/export-release.mjs --workflow taoci-outreach` 产出 `~/Desktop/Web_related/taoci-outreach/`，manifest 自包含快照
2. 再次导出生成 `taoci-outreach_1/`，不覆盖旧目录（AC-R01~R03）
3. GUI `POST /api/export-release` 与 CLI 目录结构一致，`export_entry=gui`（AC-R01b/R05）
4. 侧栏 L1情景+L2会话（admin 加 L0用户），悬停联动展开（AC-N01~N04）
5. 记忆三页独立 `/memory/user|scenario|session` + 确认队列（AC-M01~M05）
6. 多模态 zip ingest + PDF 出站（AC-F01~F03）
7. Workflow 记忆节点 ScenarioMemoryLoad/Save、SessionMemoryLoad/Save、UserMemoryLoad 替换 TaociSessionLoad/Save
8. `npm run test:web-release` 退出码 0

## 工作逻辑

**固定顺序：Debug > Test > Dev**

1. **Test 先行**：按 `WEB_ACCEPTANCE.md` 先写 `export-release.test.mjs`、`polar/tests/*`，再实现
2. **Dev 分阶段**：W0→W7 依赖顺序（见执行计划），每阶段 commit 前跑该阶段 AC
3. **Debug 即时**：轮内发现问题立即修，不累积到下一批
4. **RetryLoop**：外环 `max_retries=7`；默认 IDE 直接实现；passed = 极限目标全满足；轮间从任务书/SSoT 重新对齐，不贴 error log 当需求
5. **禁止手写网站**：只改 `_template/polar/` + PolarUI 编译脚本，发行版由 `export-release.mjs` 生成

## 用户预期体验

### 体验 1：开发者导出网站
1. 在 PolarUI 画布打开 `taoci-outreach`
2. 点击顶栏「导出网站」
3. 看到进度流；成功显示 `release_path`（如 `~/Desktop/Web_related/taoci-outreach_1`）
4. 进入该目录 `npm start` 即可独立运行，与 PolarUI 源码解耦

### 体验 2：Agent CLI 导出
1. Agent 执行 `node scripts/export-release.mjs --workflow taoci-outreach --json`
2. 解析 JSON 得到 `release_id`、`release_path`
3. 报告路径，**不**手写 Web_related 代码

### 体验 3：终端用户使用发行版
1. 用户名直登（无密码）
2. 侧栏默认隐藏，鼠标移入展开情景+会话双层
3. 首条消息「想套辞胡友财老师」→ 自动创建情景名+会话名
4. 对话后弹出记忆确认「是否保存 major=制药工程研一」
5. 顶栏下拉进入「用户记忆」「情景记忆」「会话记录」各自独立页
6. 上传 zip 简历包 → 解码利用；收到 LaTeX 生成的 PDF 附件

### 体验 4：Admin 管理
1. admin 登录见 L0 用户栏 + L1 情景 + L2 会话
2. 选中用户 guoyunyi 后仅见其情景
3. 可跨用户查看记忆（AC-M05）

## 风险

| 风险 | 缓解 |
|------|------|
| LibreChat 体量大，fork 耗时长 | W0 先最小可运行子集 + `polar/` 层；不全量复制 dev 依赖 |
| `~/Desktop/Web_related/` 路径不存在 | export 脚本 Step 0 自动 mkdir |
| LibreChat 与 POLAR 侧栏改造冲突 | 改造隔离在 `polar/`；`UPSTREAM.md` 记录 merge 策略 |
| LLM 自动命名不稳定 | 降级「新情景/新会话」，不阻塞聊天（AC 注意 #6） |
| zip bomb | ingest 限制 50MB + 100 文件 |
| preflight 依赖 PolarPrivate/xelatex 不可用 | 测试用 mock；preflight partial 不阻塞 compile-only 路径 |
| R3 记忆节点与 taoci 图切换破坏现有测试 | 保留 TaociSessionLoad/Save 兼容层至 R3 测试绿后再删 |
| Hub Alignment 覆盖率不足 | 本方案 12 子任务均带 `[SSoT:PolarUI/Rx/Feature]` 三段引用 |
