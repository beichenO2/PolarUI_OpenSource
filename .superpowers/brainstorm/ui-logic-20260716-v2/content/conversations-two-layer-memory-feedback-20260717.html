最终交互骨架：多对话 + 两层记忆 + Agent 运行反馈

对话负责交互，Context 负责复用，Workflow 负责 FSM。Web 只把两层记忆和局部对话历史组装给 Workflow，不把 Conversation 或 Checkpoint 伪装成额外记忆层。

Input + Conversation

当前用户输入、局部对话历史和 Workflow 输出，为两类记忆提取提供证据。

→
用户记忆跨所有情景
“是对用户的建模，能揭示用户的习惯、特点、taste”
例如：偏好直接讨论产品原则；不喜欢人为状态机操作。
情景记忆仅当前 Context
“是对本情景的建模，是本情景的本质信息；对之后处理具体问题有持续性帮助或约束”
例如：本情景中 Stage 只是 Workflow 的只读投影。
