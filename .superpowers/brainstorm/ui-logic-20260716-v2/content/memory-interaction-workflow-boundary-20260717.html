纠正后的产品边界：Web UI 管交互与记忆，Workflow 管 FSM

Context 的唯一使命是形成可复用的公共记忆范围，减少重复输入。Stage 不再拥有 URL、对话或输入，只是 Workflow 可选择返回的一种状态投影。

Context = 公共记忆范围事实、材料、约束与历史成果在同一情景内复用。
Conversation = 交互通道用户从任何对话输入，Web 组装 Context Memory 后交给 Workflow。
Stage = Workflow 内部状态FSM 自行迁移；Web 只显示状态，不选择、不路由、不绑定对话。
